github pages的理解和使用心得

底下步驟，是建立兩條獨立branch，一條是專案用，一條是github pages用

  1. 在github建立專案，例如叫project01

  2. 在本地mkdir project01 && cd project01
  3. git init
  4. git remote add origin git@github.com:charlie388/project01.git
  5. 建立專案相關檔案
  6. git add .
  7. git commit -m "init commit"
  8. git checkout -b main
  9. git push -u origin main
  10. 檢查專案相關檔案是否上傳到github在main branch

  11. cd .. && rm -rf project01

  12. 在本地mkdir project01 && cd project01
  13. git init
  14. git remote add origin git@github.com:charlie388/project01.git
  15. 建立網頁相關檔案
  16. git add .
  17. git commit -m "init commit"
  18. git checkout -b gh-pages
  19. git push -u origin gh-pages
  20. 檢查網頁相關檔案是否上傳到github在gh-pages branch

  21. git fetch orign，更新之前main branch
  22. git checkout -t origin/main

github pages會指定branch，當該branch commit時，來觸發某個目錄(/或/docs)下檔案轉換或複製到https://[user].github.io/[repo name] domain。

此轉換動作，每個小時限制10次，可在Actions頁面看到轉換的進度。

因此若工作用branch和github pages指定branch相同，當工作用branch commit時，同時會觸發github pages指定branch轉換。

為了減少github pages的轉換次數，因此需要將工作用branch和github pages指定branch分開。

npm套件gh-pages，可以輔助這件工作
